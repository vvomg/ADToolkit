# AWX as-Code — IVA Mail Ansible

Этот каталог содержит декларативные ресурсы AWX (организации, типы учётных данных,
проект, инвентарь, шаблоны задач), применяемые через коллекцию `awx.awx`.

---

## Предварительные требования

- Python >= 3.9
- `ansible-core` >= 2.15
- Коллекция `awx.awx` >= 23.x

---

## 1. Установка коллекции

```bash
ansible-galaxy collection install awx.awx
```

Или с указанием версии:

```bash
ansible-galaxy collection install awx.awx:>=23.0.0
```

---

## 2. Переменные окружения

Перед запуском экспортируйте переменные подключения к AWX:

```bash
export AWX_HOST=http://10.3.6.100:8080
export AWX_USERNAME=admin
export AWX_PASSWORD=<пароль из AWX UI>
```

> **Важно**: Не передавайте пароль через аргументы командной строки и не
> сохраняйте его в файлах репозитория.

---

## 3. Применение конфигурации

```bash
ansible-playbook awx/as-code/apply-awx-config.yml
```

Плейбук применяет ресурсы в правильном порядке:
1. Организация
2. Типы учётных данных
3. Проект
4. Инвентарь
5. Шаблоны задач

---

## 4. Структура каталога

```
awx/as-code/
├── README.md                  # Этот файл
├── apply-awx-config.yml       # Главный плейбук
├── organization.yml           # Ресурс: Организация
├── credential_types.yml       # Ресурс: Пользовательские типы учётных данных
├── credentials_template.yml   # Шаблон структуры учётных данных (без секретов)
├── project.yml                # Ресурс: Проект (Git)
├── inventory.yml              # Ресурс: Инвентарь
├── job_templates.yml          # Ресурс: Шаблоны задач
└── survey_spec.yml            # Спецификация Survey для шаблонов задач
```

---

## 5. Учётные данные

Реальные значения учётных данных (SSH-ключи, пароли) вводятся **только в AWX UI**:
`Credentials → <имя> → Edit`.

Файл `credentials_template.yml` содержит только структуру (без секретов) и
предназначен для документирования, а не для автоматического применения.

---

## 6. Идемпотентность

Все задачи используют `state: present` — повторный запуск безопасен и обновляет
только изменившиеся поля.
