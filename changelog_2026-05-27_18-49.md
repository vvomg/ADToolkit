| Компонент | Роль |
|---|---|
| **ansible-runner** | Запуск плейбуков из FastAPI backend |
| **ansible.posix** | `mount`, `sysctl`, `authorized_key` |

| **community.general** | Утилиты: `ini_file`, `dict_kv` |

### Плейбуки (порядок выполнения)

```
00-bootstrap            → подготовка всех хостов
01-postgres-nfs         → PostgreSQL + NFS сервер
02-backends-install     → установка бэкендов (параллельно)
02-license-request      → запрос лицензии (run_once на be1)
[APPROVAL]              → ожидание загрузки license.txt
02-license-install      → установка лицензии + рестарт нод (serial: 1)
03-frontends            → установка фронтендов (параллельно)
04-haproxy              → балансировщик (Jinja2 → haproxy.cfg)
05-monitoring           → Prometheus + Grafana + Graylog
06-backup-config        → pg_dump cron + rsync NFS + git config-repo
07-config-dump          → snapshot конфигов с нод → YAML
08-config-apply         → применение YAML конфигов к нодам
09-config-rollback      → git rollback конфигов
health-check            → финальный assert всех сервисов
```

### Ansible Roles

| Role | Что разворачивает |
|---|---|
| `bootstrap` | Python, pip, базовые пакеты |
| `postgres` | PostgreSQL, `pg_hba.conf`, `postgresql.conf` |
| `nfs_server` | NFS сервер, `/etc/exports` |
| `nfs_client` | NFS монтирование на бэкендах |
| `ivamail_common` | Установка пакета IVA Mail (4 стратегии: URL / controller / server_same / server_per_node) |
| `ivamail_backend` | Запуск в режиме `--backend`, db_setup, cluster_config |
| `ivamail_frontend` | Запуск в режиме `--frontend`, cluster_config |
| `ivamail_license` | CMD: LicenseRequest → LicenseInstall |
| `ivamail_config` | Dump/Apply/Rollback конфигов через CMD |
| `haproxy` | HAProxy (tcp mode: IMAP 143, SMTP 25; http: 80/443) |
| `node_exporter` | Prometheus Node Exporter на всех нодах |
| `monitoring` | Prometheus + Grafana + Graylog + OpenSearch |
| `backup` | pg_dump cron, rsync NFS backup, git config-repo |
