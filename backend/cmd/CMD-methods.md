# Интерфейс командной строки
### Синтаксис
Название команды передаётся в виде строки без кавычек, далее через пробелы следуют параметры - текстовые представления JSON-объектов. Необязательные параметры указаны в описаниях ниже внутри круглых скобок `()`. Вместо массива отдельных ключей `["key1","key2","key3"]` для указания пути к настройке модуля или объекта может использоваться строка вида `"key1.key2.key3"`

### Аутентификация
CMD модуль поддерживает следующие методы аутентификации: LOGIN, PLAIN, APOP, CRAM-MD5, DIGEST-MD5. Аутентификация в CMD модуле может быть отключена для соединений на loopback адрес сервера, для этого сервер необходимо запустить с параметром `--NoAuthCmdLocalhost`.

### Общие

#### `QUIT`
Заканчивает сессию и закрывает соединение.

#### `SHUTDOWN`
Инициирует остановку сервера.

#### `INLINE`
Переключает режим выдачи объектов в результатах команд между многострочным и компактным.

#### `EXT`
Переключает режим выдачи объектов в результатах команд между стандартным и расширенным.

#### `NOTIMEOUT`
Устанавливает тайм-аут чтения в соединении в 365 дней.

#### `NOESCAPE`
Переключает флаг экранирования служебных символов в выводе последующих команд.

#### `HELP ["search"]`
Выводит список команд в возможностью фильтрации.

#### `PING`
Отвечает фиксированным сообщением.

#### `RaiseMem`
Приводит к неисправимой ошибке при выполнении программы с генерацией системного сигнала `SIGSEGV`. Необходимо для проверки создания отладочных образов памяти выполняемой программы (`core dump`).

#### `SchemaDump "type" (false)`
Выдаёт описания полей для настроек по схеме _type_. Если _type_ задан пустой строкой, выдаётся список известных схем. Если дополнительно задан параметр `false`, то описания объектов в схеме не содержат прав доступа.

#### `ModulesList`
Выдаёт список инициализированных модулей.

#### `MailModulesList`
Выдаёт список инициализированных модулей обработки почты.

#### `MailQueueList ("module_name")`
Выдаёт информацию о почтовой очереди модуля по его имени (общую очередь при отстутствии имени или пустом имени).

#### `MailBatchList "module_name" (true)`
Выдаёт информацию об ожидающих/активных блоках доставки очереди почтового модуля по его имени.

#### `MailQueueInfo queueID`
Выдаёт информацию о сообщении queueID.

#### `MailQueueFail queueID ("error message")`
Удаляет из очереди сообщение queueID, проставляя адресам опциональную ошибку.

#### `ModuleReadConfig "module_name" (true|false)`
Выдаёт словарь с настройками модуля. Если дополнительно передан параметр `true`, то словарь с настройками содержит описания полей. Если дополнительный параметр задан `false`, то словарь не содержит настроек по умолчанию.

#### `ModuleUpdateConfig "module_name" [keysToDelete] [keyValPairs]`
Обновляет настройки модуля. `keysToDelete` - масив ключей для удаления из настроек (вместо них будут использоваться значения по умолчанию), `keyValPairs` - массив вида `ключ1,значение1,ключ2,значение2,...`, по которому значения указанных ключей заменятся на новые.

#### `MODULESetLogLevel "module_name" value`
Задаёт детализацию журналирования для модуля.

#### `ModuleGetSetting "module_name" "key"|["key1","key2",...]`
Выдаёт объект настроек "_key_" модуля с именем "_module_name_".

#### `ModuleSetSetting "module_name" "key"|["key1","key2",...] object`
Устанавливает в значение _object_ объект настроек "_key_" модуля с именем "_module_name_".

#### `ModuleDelSetting "module_name" "key"|["key1","key2",...]`
Удаляет объект настроек "_key_" модуля с именем "_module_name_", возвращая значение по умолчанию.

#### `DigestList`
Выдаёт список доступных методов хэширования.

#### `CipherList`
Выдаёт список доступных методов шифрования.

#### `KDFList`
Выдаёт список доступных методов преобразования паролей в ключи.

#### `ECList`
Выдаёт список названий доступных эллиптических кривых.

#### `ThreadsList ("filter")`
Выдаёт список активных нитей исполнения. Если задан дополнительный строковый параметр, то выводится информация только о тех нитях, чьё описание содержит эту строку.

#### `ConnectionsList ("module" ("filter"))`
Выдаёт список активных сетевых соединений. Если задано имя модуля, то выводится информация только о соединениях модуля. Если задан дополнительный строковый параметр, то выводится информация только о соединениях, чьё описание содержит эту строку.

#### `LogsList ("log_type")`
Возвращает список лог файлов с размерами в байтах. Если тип логов не указан или указана пустая строка - возвращает список основных лого сервера. Тип лога может быть указан как "Mailbox", "Auth", "Settings", тогда команда вернет список логов мейлбоксов, аутентификаций и настроек соответственно.

#### `LogsRemove "log_type" ["log_name1" (,"log_name2", ...)]`
Удаляет лог файлы указанного типа с именами (должны быть указаны без расширения), которые присутствуют в массиве. Тип логов либо пустая строка, тогда удаляются основные логи сервера, либо одна из строк "Mailbox", "Auth", "Settings", тогда команда удаляет логи мейлбоксов, аутентификаций и настроек соответственно.

#### `maintenance "*" | "domain.dom" | "user@domain.dom" ("SubtaskName")`
Запускает задачу Maintenance для всего сервера, или для отдельного домена, или для отдельного аккаунта.
Задача Maintenance состоит из подзадач:
1) CleanTrash - очистка папки Корзины от старых сообщений в аккаунте
2) CleanSpam - очистка папки Спам от старых сообщений в аккаунте
3) CleanMail - удаление старых сообщений из всех папок в аккаунте
4) RestructureMail - перемещение старых сообщений из папок INBOX и Отправленные в подпапки Архивы с раскладкой по кварталам года
5) ArchiveMail - перемещение старых сообщений из всех папок в подпапки Архивы с сохранением оригинальной структуры папок
6) CleanCalendars - удаление старых старых календарных событий из всех календарей в аккаунте
7) RestructureCalendar - перемещение старых календарных событий из главного календаря в подпапки Архивы с раскладкой по кварталам года
8) CleanDomain - удаляет из домена объекты, имеющие в настройках ключ DeleteMeAfter со значением отметки времени, которая меньше текущего времени
9) RepairServer - ничего не делает, пока не придумали чем заняться
10) RepairDomain - ничего не делает, пока не придумали чем заняться
11) RepairAccount - ничего не делает, пока не придумали чем заняться

Если имя подзадачи указано, то выполняется только она; иначе выполняются подзадачи 1-8

#### `FeatureFlagSet ["flagName",...] enabled`
Включает или выключает временно ограничиваемый для разработки функционал и возвращает текущее состояние.



### Работа с доменами

#### `DOMAINSLIST ("filter"|true|false|null )`
Выдаёт список зарегистрированных доменов в виде словаря с числовыми ключами: ключ - `UID` домена, значение - основное имя домена. Если указать фильтр поиска __filter__, то будет произведена фильтрация по наличию подстроки как в основном доменном имени, так и в псевдониме. Если вместо фильтра поиска указать булево значение `false` или `true`, то будет выведен подробный список с типом, всеми доступными именами и псевдонимами и флагами (при `true` будут также выведены и проверочные хэш-суммы). Параметр `null` может быть использован для выдачи информации только о кластерных доменах.

#### `DOMAINCREATE "string" (null)`
Инициирует создание домена с основным именем "_string_". Если указан необязательный параметр `null`, то создаётся кластерный домен.

#### `DOMAINREMOVE "string"|domUID`
Инициирует удаление домена с именем "_string_" или числовым идентификатором `domUID`.

#### `DOMAINRENAME "string1"|domUID "string2"`
Инициирует переименование (смену основного имени) домена с именем "_string1_" или числовым идентификатором `domUID` в "_string2_"

#### `DOMAINGETID "string"`
Выдаёт UID домена по его имени "_string_".

#### `DOMAINLISTNAMES "string"|domUID`
Выдаёт список всех имён домена по его имени "_string_" или числовому идентификатору `domUID`.

#### `DOMAINADDNAME "string1"|domUID "string2" (flag)`
Инициирует добавление имени "_string2_" с необязательным (его значение по умолчанию равно 255) флагом _flag_ домену с именем "_string1_" или числовым идентификатором `domUID`.

#### `DOMAINDELNAME "string1"`
Инициирует удаление имени "_string1_" у домена с таким именем.

#### `DOMAINUSENAME "string1"`
Имя "_string1_" должно принадлежать домену и по выполнении команды становится основным (первым в списке присвоенных домену).

#### `DOMAINMOVENAME "string1"|domUID "string2"`
Инициирует атомарный перенос имени "_string2_" домену с именем "_string1_" или числовым идентификатором `domUID`.

#### `DOMAINGETFLAGS "string"`
Выдаёт флаги применения для имени домена "_string_".

#### `DOMAINSETFLAGS "string" flag`
Задаёт флаги применения _flag_ для имени домена "_string_".

#### `DOMAINREADCONFIG "string"|domUID (true|false)`
Выдаёт объект настроек домена с именем "_string_" или числовым идентификатором `domUID`. Если дополнительно передан параметр `true`, то словарь содержит описания полей. Если дополнительный параметр задан `false`, то словарь не содержит настроек по умолчанию.

#### `DOMAINGETSETTING "string"|domUID "key"|["key1","key2",...]`
Выдаёт объект настроек "_key_" домена с именем "_string_" или числовым идентификатором `domUID`.

#### `DOMAINSETSETTING "string"|domUID "key"|["key1","key2",...] object`
Устанавливает в значение _object_ объект настроек "_key_" домена с именем "_string_" или числовым идентификатором `domUID`.

#### `DOMAINDELSETTING "string"|domUID "key"|["key1","key2",...]`
Удаляет объект настроек "_key_" домена с именем "_string_" или числовым идентификатором `domUID`, возвращая значение по умолчанию.

#### `DOMAINUPDATECONFIG "string"|domUID [keysToDelete] [keyValPairs]`
Обновляет настройки домена с именем "_domname_" или числовым идентификатором `domUID`, `keysToDelete` - масив ключей для удаления из настроек (вместо них будут использоваться значения по умолчанию), `keyValPairs` - массив вида `ключ1,значение1,ключ2,значение2,...`, по которому значения указанных ключей заменятся на новые.

#### `KerberosKeyAdd "string"|domUID "{keytab_base64}"`
Добавляет запись keytab Kerberos в домен.

#### `KerberosKeyList "string"|domUID`
Выдаёт список записей keytab Kerberos для домена.

#### `KerberosKeyRemove "string"|domUID index`
Удаляет запись keytab Kerberos из домена по индексу.

#### `DOMAINGETMULTISET "string"|domUID [keysToRequest]`
Выдаёт объект настроек домена с именем "_string_" или числовым идентификатором `domUID` для ключей из массива `keysToRequest` (могут быть указаны ключи только верхнего уровня).

#### `DOMAINHASINLIST "domname"|domUID "key"|["key1","key2",...] object`
Проверяет присутствие значения _object_ в списке объекта настроек "_key_" домена с именем "_domname_" или числовым идентификатором `domUID`.

#### `DOMAINADDTOLIST "domname"|domUID "key"|["key1","key2",...] object`
Добавляет значение _object_ в список объекта настроек "_key_" домена с именем "_domname_" или числовым идентификатором `domUID`.

#### `DOMAINDELINLIST "domname"|domUID "key"|["key1","key2",...] object`
Удаляет значение _object_ из списка объекта настроек "_key_" домена с именем "_domname_" или числовым идентификатором `domUID`.

#### `DOMAINGETWEBACCESSURL "domname"|domUID`
Возвращает строковое значение URL-адреса веб сервера в домене с именем "_domname_" или числовым идентификатором `domUID`.

#### `DOMAINRUNHOOK "domname"|domUID [keys] oldObject (newObject)`
Выполняет обратный вызов, установленный для ключа `keys` настроек домена с именем "_domname_" или числовым идентификатором `domUID`.
Передаются старое значение настройки по указанным ключам `oldObject` (`null`, если значения не было) и новое значение `newObject`
(может отсутствовать при удалении настройки). Метод предназначен исключительно для использования в кластерном протоколе.

#### `DomainStaticPassword "domname"|domUID ("password")`
Устанавливает пароль для доступа на члены статического кластера (распределённого домена). Если пароль не задан, то настройка удаляется (пароль сбрасывается и доступ к членам стаического кластера невозможен).

#### `DOMAINSDEFAULTS (true|false) (null)`
Выдаёт объект настроек по умолчанию для локальных доменов. Если дополнительно передан параметр `true`, то словарь содержит описания полей. Если дополнительный параметр установлен в `false`, то выдаётся объект настроек без учёта жёстко заданных в программе. Если дополнительно задан параметр `null`, то выдаются значения по умолчанию для кластерных доменов.

#### `DOMAINSGETDEFAULT "key"|["key1","key2",...] (null)`
Выдаёт объект настроек по умолчанию для локальных доменов "_key_". Если дополнительно задан параметр `null`, то выдаётся значение по умолчанию для кластерных доменов.

#### `DOMAINSSETDEFAULT "key"|["key1","key2",...] object (null)`
Устанавливает в значение _object_ объект настроек по умолчанию для локальных доменов "_key_". Если дополнительно задан параметр `null`, то изменяется значение по умолчанию для кластерных доменов.

#### `DOMAINSDELDEFAULT "key"|["key1","key2",...] (null)`
Удаляет объект настроек "_key_" из настроек локальных доменов по умолчанию, возвращая значение по умолчанию. Если дополнительно задан параметр `null`, то изменяется значение по умолчанию для кластерных доменов.

#### `DOMAINSUPDATEDEFS keysToDelete keyValPairs (null)`
Обновляет настройки по умолчанию для локальных доменов, `keysToDelete` - масив ключей для удаления из настроек (вместо них будут использоваться значения по умолчанию),
`keyValPairs` - массив вида `ключ1,значение1,ключ2,значение2,...`, по которому значения указанных ключей заменятся на новые. Если дополнительно задан параметр `null`, то изменяются значения по умолчанию для кластерных доменов.

### Работа с объектами в доменах
Первым параметром всегда является либо имя, либо `UID` домена.

#### `IndexGetEntry "domname"|domUID ("objname"|accUID)`
Выдаёт запись из индекса доменов или индекса объектов в домене.

#### `OBJECTSLIST "domname"|domUID` ("filter"|true|false) (start limit ([fields]))
Выдаёт список зарегистрированных в домене объектов в виде словаря с числовыми ключами: ключ - `UID` объекта, значение - основное имя объекта. Если указать фильтр поиска _filter_, то будет произведена фильтрация по наличию подстроки как в основном имени объекта, так и в псевдониме. Если вместо фильтра поиска указать булево значение `false` или `true`, то будет выведен подробный список с типом, всеми доступными именами и псевдонимами и флагами (при `true` будут также выведены и проверочные хэш-суммы). Если заданы числовые параметры _start_ и _limit_, то выдача будет ограничена элементами с `UID` не менее _start_ и общим количеством не более _limit_. Дополнительно можно указать массив ключей, значения которых должны быть включены в информацию об объекте, например `["RealName", "StorageSize"]`.

#### `ACCOUNTCREATE "domname"|domUID "accname" (settingsDict|settingsArray)`
Инициирует создание объекта "учётная запись" с основным именем "_accname_" в домене с именем "_domname_" или имеющим UID _domUID_. При праве `r` пользователю доступно чтение файла или просмотр содержимого папки.
При праве `w` пользователю доступна запись или удаление файла или каталога.

`anyone` для указания любого аккаунта или анонимного доступа.

Задаёт имя владельца группы. Владелец имеет возможность управлять группой, не имея административных прав.

### Работа с мейлбоксами
Первым параметром всегда является либо имя, либо `UID` домена.
Вторым параметром всегда является либо имя, либо `UID` аккаунта.

#### `MAILBOXESLIST "domname"|domUID "accname"|accUID ("class") (optMode (optParent))`
Выдаёт словарь всех мейлбоксов аккаунта. Ключи словаря  - полные имена мейлбоксов, построенные в зависимости от необязательного параметра `optMode`:
- 0 - имена в UTF-8 (по умолчанию)
- 1 -  имена в модифицированной кодировке UTF-7
- 2 - имена поддиректорий в директории аккаунта
Если указан необязательный cтроковый параметр "_class_", то в словарь будут включены только мейлбоксы данного класса.
Если указан необязательный числовой параметр `optParent`, то листинг будет произведён только для папок вложенных в папку с UID `optParent` (0 для всех папок аккаунта).

#### `MAILBOXCREATE "domname"|domUID "accname"|accUID "fullName" ("class")`
Создаёт новый мейлбокс в аккаунте с именем "_fullName_" и классом "_class_" (если данный параметр не существует, то используется класс "_mail_"). Все варианты имени `Inbox` в корне приводятся к верхнему регистру `INBOX`.
Поддерживаемые значения `class`: `mail`, `calendar`, `contacts`, `notes`, `tasks`.

#### `MAILBOXINSTALL "domname"|domUID "accname"|accUID "filename" ("class")`
Регистрирует в аккаунте мейлбокс из директории с именем "_filename_" в директории аккаунта и классом "_class_" (если данный параметр не существует, то используется класс "_mail_"). При конфликте имени или UID, разобранных из имени директории, возможно их изменение.
Поддерживаемые значения `class`: `mail`, `calendar`, `contacts`, `notes`, `tasks`.

#### `MAILBOXRENAME "domname"|domUID "accname"|accUID "oldFullName" "newFullName"`
Переименовывает мейлбокс с полным именем  "_oldFullName_" в имя "_newFullName_". При переименовании мейлбокс может изменить позицию в иерархии с ожидаемыми ограничениями (например, нельзя переименовать `a/b` в `a/b/c` и наоборот).

#### `MAILBOXREMOVE "domname"|domUID "accname"|accUID "fullName"`
Удаляет мейлбокс с полным именем  "_fullName_" вместе со всеми вложенными мейлбоксами.

#### `MboxMessageInfo "dom.name"|domUID "accname"|accUID "mbox"|mbxUID msgUID`
Выдаёт информацию о сообщении с msgUID в мейлбоксе с именем "_mbox_" или ID _mbxUID_ аккаунта с именем "_accname_" или ID _accUID_ домена с именем "_dom.name_" или ID _domUID_.

#### `MboxMessageRemove "dom.name"|domUID "accname"|accUID "mbox"|mbxUID msgUID ("dst_dom.name"|dst_domUID "dst_accname"|dst_accUID) ("dst_mbox"))`
Удаляет сообщение с msgUID в мейлбоксе с именем "_mbox_" или ID _mbxUID_ аккаунта с именем "_accname_" или ID _accUID_ домена с именем "_dom.name_" или ID _domUID_.
Если заданы "dst_dom.name" и "dst_accname", то вместо удаления делает перемещение сообщения в этот аккаунт. Если не задан "dst_mbox", то создаст новую папку с текущем временем в имени. 

#### `MailboxGetACL "domname"|domUID "accname"|accUID "fullName"`
Выдаёт ACL (Access Control List, или список управления правами) мейлбокса в виде словаря.
Мейлбокс может быть задан в виде `~user@domain/mailbox`
Для выдачи корневого ACL аккаунта нужно указать имя мейлбокса как пустую строку для себя, или `~user@domain/` для другого аккаунта.

#### `MailboxUpdateACL "domname"|domUID "accname"|accUID "fullName" "identifier" ("modifier")`
Меняет элемент _identifier_ в ACL мейлбокса (_identifier_ это не обязательно аккаунт!).
modifier может начинается с символов '+' и '-', тогда новые права соответственно добавляются или удаляются из старых. Если все права удалены, то _identifier_ удаляется из ACL.
Если modifier начинается с '*', то устанавливаются все права.
Если modifier пустая строка или не указан, то _identifier_ удаляется из ACL.
Если modifier начинается с '#' (особый случай), то identifier сравнивается с элементами ACL через маршрутизатор, и совпавшие элементы удаляются (может удалить сразу несколько идентификаторов из ACL, если они заданы как алиасы).
Мейлбокс может быть задан в виде `~user@domain/mailbox`

#### `MailboxGetMyRights "domname"|domUID "accname"|accUID "fullName"`
Выдаёт права аккаунта на мейлбокс в виде строки. Мейлбокс может быть задан в виде `~user@domain/mailbox`

#### `CalendarGetACL "domname"|domUID "accname"|accUID "fullName"`
#### `CalendarUpdateACL "domname"|domUID "accname"|accUID "fullName" "identifier" ("modifier")`
#### `CalendarGetMyRights "domname"|domUID "accname"|accUID "fullName"`
Аналогичны предыдущим трем вызовам, но для дополнительных календарных прав доступа.

#### `MboxGetInfo "domname"|domUID "accname"|accUID "mailboxname"|mboxUID ("settingskey")`
Выдаёт значение метаданных почтовой папки _mailboxname_ по ключу _settingskey_. Если ключ _settingskey_ не задан, то возвращается весь словарь с информацией о папке.

#### `MboxSetInfo "domname"|domUID "accname"|accUID "mailboxname"|mboxUID "settingskey" (object)`
Устанавливает метаданных почтовой папки _mailboxname_ по ключу _settingskey_ в значение _object_. Если значение _object_ не указано, ключ _settingskey_ будет удалён из метаданных.

#### `ObjectGetInfo "domname"|domUID "accname"|accUID "settingskey"`
Выдаёт значение метаданных объекта по ключу _settingskey_.

#### `ObjectSetInfo "domname"|domUID "accname"|accUID "settingskey" (object)`
Устанавливает метаданных объекта по ключу _settingskey_ в значение _object_. Если значение _object_ не указано, ключ _settingskey_ будет удалён из метаданных.

#### `ObjectIncInfo "domname"|domUID "accname"|accUID "settingskey" "delta" "limitkey"`
Увеличивает значение счётчика (обычного или ограниченного по времени, в зависимости от типа существующего значения и типа предела) в метаданных объекта по ключу _settingskey_ на значение _delta_ в соответствии с пределом, установленным в настройках объекта по ключу _limitkey_.

#### `MessageGetAttribute "dom.name"|domUID "accname"|accUID "mboxName" msgID "attrName"`
Выдаёт значение атрибута `attrName` сообщения с идентификатором `msgID`
в почтовом ящике `mboxName" аккаунта `"accname"|accUID` домена `"dom.name"|domUID`.
Атрибут "." означает все атрибуты (корневой элемент).

#### `MessageSetAttribute "dom.name"|domUID "accname"|accUID "mboxName" msgID "attrName" value`
Устанавливает в значение `value` атрибут `attrName` сообщения с идентификатором `msgID`
в почтовом ящике `mboxName" аккаунта `"accname"|accUID` домена `"dom.name"|domUID`.
Атрибут "." означает все атрибуты (корневой элемент).

### Работа с административными правами

#### `RightsList (true)`
Выдаёт список административных прав, доступных запрашивающему администратору. При указании дополнительного параметра `true`, список прав выдаётся в виде вложенных словарей согласно иерархии прав.

#### `AccessDump "dom.name"|domUID|null`
Выдаёт список зарегистрированных административных ролей (именованных наборов прав) и администраторов в домене с именем "_dom.name_" или числовым идентификатором `domUID`. Если в качестве идентификатора домена передана пустая строка или число 0, то зачитываются данные на уровне сервера. Если вместо идентификатора домена передан `null`, то зачитываются данные на уровне кластера (на уровне сервера, если сервер запущен вне кластерной конфигурации).

#### `RolesList "dom.name"|domUID|null`
Выдаёт список административных ролей (именованных наборов прав) в домене с именем "_dom.name_" или числовым идентификатором `domUID`. Если в качестве идентификатора домена передана пустая строка или число 0, то зачитываются роли на уровне сервера. Если вместо идентификатора домена передан `null`, то зачитываются роли на уровне кластера (на уровне сервера, если сервер запущен вне кластерной конфигурации).

#### `RoleCreate "dom.name"|domUID|null "role" ([rights] (true))`
Создаёт административную роль для домена или всего сервера/кластера (аналогично параметрам команды `RolesList`) с именем _role_ и набором прав из массива _rights_. Опциональный булевый флаг предназначен для подавления оповещений о смене прав в кластере.

#### `RoleRemove "dom.name"|domUID|null "role" (true)`
Создаёт административную роль домена или всего сервера/кластера (аналогично параметрам команды `RolesList`) с именем _role_. Опциональный булевый флаг предназначен для подавления оповещений о смене прав в кластере.

#### `RoleGet "dom.name"|domUID|null "role"`
Выдаёт список административных прав для роли _role_ в домене или во всём сервере/кластере (аналогично параметрам команды `RolesList`).

#### `RoleUpdate "dom.name"|domUID|null "role" [rghtsDel] [rghtsAdd] (true)`
Изменяет административную роль для домена или всего сервера/кластера (аналогично параметрам команды `RolesList`) с именем _role_, удаляя из неё права из массива _rghtsDel_ и добавляя права из массива _rghtsAdd_. Опциональный булевый флаг предназначен для подавления оповещений о смене прав в кластере.

#### `AdminsList "dom.name"|domUID|null`
Выдаёт список административных аккаунтов в домене с именем "_dom.name_" или числовым идентификатором `domUID`. Если в качестве идентификатора домена передана пустая строка или число 0, то зачитывается список администраторов на уровне сервера. Если вместо идентификатора домена передан `null`, то зачитывается список администраторов на уровне кластера (на уровне сервера, если сервер запущен вне кластерной конфигурации).

#### `AdminCreate "dom.name"|domUID|null "user@domain" ([rights] (true)`
Для домена или всего сервера/кластера (аналогично параметрам команды `AdminsList`) даёт административные права из массива _rights_ пользователю _user\@domain_. Опциональный булевый флаг предназначен для подавления оповещений о смене прав в кластере.

#### `AdminRemove "dom.name"|domUID|null "user@domain" (true)`
Удаляет администратора _user\@domain_ из списка администраторов домена или всего сервера/кластера (аналогично параметрам команды `AdminsList`). Опциональный булевый флаг предназначен для подавления оповещений о смене прав в кластере.

#### `AdminInfo "dom.name"|domUID|null "user@domain"`
Выдаёт список административных прав и ролей для для роли администратора _user\@domain_ в домене или во всём сервере/кластере (аналогично параметрам команды `AdminsList`).

#### `AdminUpdate "dom.name"|domUID|null "user@domain" [rghtsDel] [rghtsAdd] (true)`
Изменяет административные права и роли администратора _user\@domain_ в домене или во всём сервере/кластере (аналогично параметрам команды `AdminsList`), удаляя из неё права из массива _rghtsDel_ и добавляя права из массива _rghtsAdd_. Опциональный булевый флаг предназначен для подавления оповещений о смене прав в кластере.

#### `AdminSet "dom.name"|domUID|null "user@domain" [rights] (true) (true)`
Изменяет административные права и роли администратора _user\@domain_ в домене или во всём сервере/кластере, заменяя текущие права и/или роли на права и/или роли из массива _rights_. Первый опциональный булевый параметр задает принудительный режим: если аккаунт не является администратором и список прав в параметре команды не пуст, то пользователь будет автоматически назначен администратором, и наоборот, - если пользователь является администратором и список прав в параметре команды пуст, пользователь автоматически будет удален из списка административных аккаунтов. Второй опциональный булевый флаг предназначен для подавления оповещений о смене прав в кластере.

### Работа со справочником

В зависимости от настроек справочник может обращаться к тому или иному серверу
(postgres,ldap, ...)

#### `DirectoryFind "base_dn" "scope" "filter" ["field"(,"field"...)] "sorting" "cookie" limit`
Выдаёт список записей справочника соответствующих запросу:

* `base_dn` - базовый dn
* `scope` - тип поиска (
`base` - запись с указанным dn,
`one` - непосредственные дочерние записи для указанного dn,
`sub` - все дочерние записи для указанного dn)
* `filter` - фильтр формата `(<поле>=<значение>)`.
Значение может содержать `*` (любое число любых символов) и `?` (один любой символ).
Для объединения нескольких условий используются `(&(...)(...))` (выполнение всех условий)
или `(|(...)(...))` (выполнение хотя бы одного из условий). (м.б. пустым)
* `fields` - атрибуты для получения по каждой записи (массив строк). * - все доступные атрибуты
* `sorting` - сортировка результатов в формате `[+-]<название атрибута>`. Первый символ определяет направление сортировки. (м.б. пустым)
* `cookie` - текст, возвращенный последней записью предыдущего результата, для загрузки следующей страницы (м.б. пустым)
* `limit` - максимальное число записей в ответе (1..500, м.б. пустым, тогда = 100)

#### `DirectoryAdd "dn" {attributes}`
Добавляет запись с указанным dn и заданными атрибутами в формате json.

#### `DirectoryUpdate "dn" {changed_attributes}`
Обновляет/объединяет атрибуты записи с указанным dn заданными атрибутами в формате json.

#### `DirectoryMove "dn" "new_dn"`
Переименовывает/перемещает запись с указанным dn в запись с new_dn.

#### `DirectoryRemove "dn"`
Удаляет запись с указанным

#### `DirectoryCleanup "dn" ["min_tombstone_time"]`
Удаляет записи об удаленных объектах из справочника для указанного dn и всех его дочерних элементов.
Удаляются записи только в томе с указанным dn.
Удаляются записи только по объектам, удаленным ранее `min_tombstone_time`.
При отсутствии `min_tombstone_time` используется текущее время.
При отсутствии `dn` используется "".

#### `DomainsToDirectory ["dom1.name"|dom1UID,"dom2.name"|dom2UID,...]`
Выполняет синхронизацию справочника для указанных доменов.
При пустом списке доменов синхронизируются все домены.

### Клиент LDAP

#### `LdapClientSearch "base_dn" "scope" "deref" "filter" ["field"(,"field"...)] "attrsonly|all" size.limit time.limit "sorting" "cookie" "server"`

Выдаёт список записей `server`, соответствующих запросу:

* `base_dn` - базовый dn
* `scope` - тип поиска (
`base` - запись с указанным dn,
`one` - непосредственные дочерние записи для указанного dn,
`sub` - все дочерние записи для указанного dn)
* `deref` - тип разыменования ссылок (
`never` - никогда,
`always` - всегда,
`searching` - разыменовывать при определении дочерних объектов,
`finding` - разыменовывать при определении базового объекта)
* `filter` - фильтр формата `(<поле>=<значение>)`.
Значение может содержать `*` (любое число любых символов) и `?` (один любой символ).
Для объединения нескольких условий используются `(&(...)(...))` (выполнение всех условий)
или `(|(...)(...))` (выполнение хотя бы одного из условий). (м.б. пустым)
* `fields` - атрибуты для получения по каждой записи (массив строк)
* `attrsonly|all` - выдавать только названия атрибутов (`attrsonly`) или также их значения (`all`)
* `size.limit` - максимальное число записей в ответе (1..500, м.б. пустым, тогда = 100)
* `time.limit` - максимальное время выполнения запроса LDAP-сервером
* `sorting` - сортировка результатов в формате `[+-]<название атрибута>`. Первый символ определяет направление сортировки. (м.б. пустым)
* `cookie` - текст, возвращенный последней записью предыдущего результата, для загрузки следующей страницы (м.б. пустым). Игнорируется при `size.limit` == 0
* `server` - URI с параметрами доступа к серверу вида `ldap://user:pass@host:port`.

import - сохраняет событие, заменяя существующее, если есть, без выполнения дополнительных действий (уведомления, обновление dtstamp, ...).

### Работа со списками адресов

#### `AddressListCheck "IP address" "address list as text"`
Проверяет наличие адреса в списке.
Результат (число)
1 = адрес включён в список
0 = не включён
-1 = исключён
-2 = ошибка: неизвестный вложенный список
-3 = ошибка: слишком глубокая вложенность
Также может вернуть ошибку "500 Bad format for an IP address"

Примеры:
```
AddressListCheck "10.0.0.1" "#NATed\n!10.0.0.2"
AddressListCheck "178.177.53.210" "#@*iva.ru"
```

#### `AddressListGet ("name")`
Если задан параметр "name", то выдаёт список адресов с этим именем в виде строки.
Если не задан, то выдаёт все именованные списки адресов в виде словаря.

Есть специальные имена, которые не выдаются в общем списе именованных адресов, но которе можно получить и изменить по-отдельности:
   - `$BLOCKED`     - "заблокированные" - то, что раньше было "запрещённые" адреса.
   - `$UNBLOCKABLE` - "неблокируемые" или "белый список".
    -`$DEBUG`       - "адреса под отладкой"

Пример:
`AddressListGet "LAN"`

#### `AddressListUpdate param1 ( param2 )`
Обновляет списки адресов.
param1 = словарь - заменяет все именованные списки содержимым словаря
param1 = строка, param2 = строка - заменяет содержимое списка с именем param1 на содержимое строки param2
param1 = строка, param2 = null - удаляет список с именем param1

Пример:
`AddressListUpdate "LAN_2" "10.1.1.0/8 ; сегмент2\n!10.1.1.2 ; исключение"`


### Работа с настройками федеративного доступа

#### `FederatedSharingList ("name")`
Если задан параметр "name", то выдаёт единицу настроек с этим именем в виде словаря.
Если не задан, то выдаёт все единицы настроек в виде словаря.

#### `FederatedSharingUpdate param1 ( param2 ( param3 ))`
Обновляет настройки федеративного доступа.
param1 = словарь - заменяет все настройки
param1 = строка, param2 = словарь - заменяет содержимое единицы настроек с именем param1 на param2
param1 = строка, param2 = null - удаляет единицу настроек с именем param1
param1 = строка, param2 = строка, param3 = словарь - заменяет в единице настроек с именем param1 элемент с именем param2 на объект param3
param1 = строка, param2 = строка, param3 = null - удаляет в единице настроек с именем param1 элемент с именем param2


### Работа с планировщиком задач

### `TaskSchedule Param1 "Param2" "Param3" ("Param4" ("Param5"))`
Добавляет задание в расписание
Param1 = строка с идентификатором объекта, который может быть представлен в одном из нескольких вариантов [ "*" | "domain.dom" | "user@domain.dom" | =objUID@=domUID ]
Param2 = строка с названием запланированного действия
Param3 = время, в формате объекта времени, на которое запланировано выполнение действия
Param4 = периодичность, в формате объекта продолжительности времени, с которой действие должно повторяться
Param5 = любой объект, который необходимо передать для выполнения действия в качестве аргумента

Примеры:
```
TaskSchedule "postmaster@ivamail.local" "WriteToLog" "\/T 2025-08-27T11:00:00.000Z" 600000000
TaskSchedule "postmaster@ivamail.cluster" "WriteToLog" "\/T 2025-08-27T11:00:00.000Z" "\/D 10m"
TaskSchedule "*" "Maintenance" "\/T 2025-09-26T01:00:00.000Z" "\/D 24h"
```

### `TaskCancel TaskUID`
Извлекает задание из расписания, в качестве параметра принимает уникальный идентификатор задания
TaskUID = уникальный идентификатор задания.

### `TasksList Param1 ("Param2" ("Param3"))`
Возвращает список запланированных заданий
Param1 = булев флаг в зависимости от которого используется либо кластерный, либо обычный планировщик (`true` - используется кластерный)
Param2 = время, в формате объекта времени, начиная с которого будет сформирован список заданий
Param3 = максимальное количество запланированных заданий в результате запроса

Пример:
```
TasksList false "\/T 2025-08-27T11:00:00.000Z" 100
```

### `HttpRequest "uri" "method" "body"`

Выполняет HTTP-запрос к указанному адресу с заданными методом и телом и возвращает ответ.

### Работа с отложенными почтовыми отправлениями

#### `DeferredDeliveryPlan "dom.name"|domUID "accname"|accUID messageUID, "delayTill" "sourceMbox" ("targetMbox")`
Планирует отложенную доставку письма на указанное время.
delayTill  - время на которое планируется доставка письма
messageUID - уникальный идентификатор письма
sourceMbox - название почтового ящика в котором находится письмо
targetMbox - название почтового ящика в который следует переложить письмо после успешной отправки (если параметр не указан письмо после доставки не сохраняется)

#### `DeferredDeliveryList "domain" "account"`
Возвращает список с информацией об отложенных почтовых отправлениях (максимум 100 элементов).

#### `DeferredDeliverySend "domain" "account" "messageId" "mode" ("deadline")`
В зависимости от значения параметра `mode`, который принимает одно из трех значений, "later", "now" и "never", соответственно:
- Модифицирует время отложенной доставки письма с указанным идентификатором.
  Параметр `deadline` в этом случае - строка в формате объекта времени на которое запланирована доставка (должно быть больше текущего).
- Осуществляет немедленную доставку отложенного письма с указанным идентификатором.
- Отменяет отложенную доставку письма с указанным идентификатором.

#### `GetStaticVar "OID"`
Возвращает объект хранимый в статической переменной, где OID - идентификатор объекта в формате "1.2.3.n"

#### `RemoteLog "message" severity`
Рассылает сообщение по протоколу bsdlog на удаленные приемники, где `message` - текст сообщения, `severity` - уровень логирования от 0 до 7.
